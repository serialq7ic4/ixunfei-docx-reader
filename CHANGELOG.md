# Changelog

All notable changes to `ixunfei-docx-reader` are documented here.

This project keeps changelog entries human-written and release-focused. Update the `Unreleased` section before tagging a new version, then move those entries under the new version heading.

## Unreleased

## v0.1.5 - 2026-07-10

### Added

- Added bilingual contribution guidelines plus GitHub issue and pull request templates with privacy-safe reporting requirements.

### Changed

- Updated the test configuration so source-checkout verification imports the working tree package instead of a previously installed wheel.

### Security

- Redacted both owner IDs and OKR IDs from `ixfdoc inspect` OKR source summaries.
- Stopped including raw OKR API payloads in remote read errors.

## v0.1.4 - 2026-07-09

### Added

- Added `ixfdoc inspect <source>` to print safe source-routing diagnostics without reading document content or exposing full remote tokens.
- Added Markdown rendering for simple docx tables, task-list items, code block language metadata, and inline rich-text URL links.

### Changed

- Local Markdown outputs written with `--out-dir` now use source-derived filenames with collision-safe suffixes instead of generic `local-markdown-N.md` names.

## v0.1.3 - 2026-07-08

### Added

- Added `ixfdoc update check` to compare the local engine version with the latest GitHub Release and print a copyable upgrade command.
- Added `ixfdoc update skills` to refresh installed Codex / Claude Code skill wrappers from the currently installed package.

### Changed

- Bumped package version to `0.1.3` for the update workflow release.

## v0.1.2 - 2026-07-08

### Added

- Added `ixfdoc read` support for i讯飞/LarkShell OKR pages, rendered as Objective / Key Result Markdown.
- Added bilingual README documentation with Chinese as the default and English in `README.en.md`.
- Added skill-first installation guidance for Codex and Claude Code.
- Added changelog requirement to the release workflow.

### Changed

- Repositioned the project as a Codex / Claude Code skill installed by default, with `ixfdoc` as the local execution engine.
- Reworked README structure to emphasize agent usage, local privacy, supported platforms, and underlying commands for debugging.
- Replaced named third-party comparison wording with generic browser-extension export tool wording.

### Changed

- Bumped package version to `0.1.2` for the OKR reader release.

## v0.1.1 - 2026-07-06

### Added

- Added `ixfdoc read --cleanup` to remove only files generated by the current read command.
- Added cleanup guidance to packaged Codex and Claude Code skills.

### Changed

- Bumped package version to `0.1.1` and published GitHub Release artifacts.

### Verified

- Verified local tests, ruff, build, smoke script, and GitHub Actions CI on macOS/Windows with Python 3.11/3.12.

## v0.1.0 - 2026-07-06

### Added

- Initial public GitHub Release.
- Added `ixfdoc` local execution engine for reading authorized i讯飞/LarkShell documents into Markdown/TSV artifacts.
- Added macOS LarkShell cookie export via local Chromium profile data and Keychain.
- Added experimental Windows LarkShell cookie export via Chromium profile data and DPAPI.
- Added `ixfdoc doctor` diagnostics without printing cookie values.
- Added remote private document reader migrated from the original local skill.
- Added Feishu/i讯飞 docx client-vars to Markdown conversion.
- Added embedded sheet expansion into TSV sidecar files.
- Added Codex and Claude Code skill installation through `ixfdoc setup skills`.
- Added GitHub Actions CI and tagged Release workflow.

### Documentation

- Added supported-platform, release, privacy, security, and error-contract documentation.
